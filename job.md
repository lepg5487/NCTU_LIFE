# 被問
```
#Proxmox是什麼？
Proxmox當初是學長架設的，只需要了解當初硬體架設的架構就能使用，像是哪個Server和哪個Switch接，當要測試時Client應該是要和哪個Switch接等等。

#能不能接受派駐和出差?
因為沒有派駐過所以還不知道, 但是可能可以從出差開始慢慢了解自已能不能接受派駐。

#為什麼想應徵MIS系統開發工程師(動機)？
想了解大公司的網路和伺服器環境。

#還有應徵哪些公司?
群創, 緯穎, 緯創, 群聯等等, 職位基本上都是MIS或是自動化測試。

#為什麼台南(高雄)(台中)工作？
因為和女朋友交往7年，她在附近工作就和她一起來這裡工作。

#Dlink合作的時候遇到什麼問題？
和Dlink合作的時候，那時候做SDN的實驗，我已經把理論上可以做的事都做完了，但是實驗就是無法成功，結果最後是因為筆電有二張網卡，一個是WiFi的一個是有線網路，封包全部都往有連上網路的WiFi那張網卡送，造成整個實驗失敗。

#大學專題有遇到什麼困難？
大學專題遇到比較困難的是，在整理和整合開放資料的時候，過程中都是使用自己電腦開發，但放到Server上去運行的時候，都不會那麼順利，需要一直檢查Server的Log來解決問題。

#你會後悔休學嗎？
不會，畢竟是做自己喜歡的事情，雖然沒辦法把喜歡的事情變成工作，但是我也學到了很多，像是做任何事都要像我當初面對學業那樣的謹慎，像考試一樣要考慮到全方面。
如果要具體一點的形容的話，就是盡量要有SOP，這樣才比較好去Debug。

#你的優缺點是什麼？ 
我覺得我的優點是，學習的速度蠻快的，因為其實上研究所的時候完全沒有網路基礎，是上了以後用1個月的時間把交大的網路課全部上過一遍，才開始有基礎的。 
我是個比較內向的人，有時候我在比較多人的場合，我會害羞不敢提出我的想法，所以最後我都是私底下和別人討論。

#你喜歡程式還是網路？
其實我二個都很喜歡，
網路方面的知識，可以讓我了解處理事情的方法有哪些，像是要和其他人確認一件重要的事情時，可以想像成使用 TCP 的三次握手(three way handshake)過程來確保彼此的認知沒有錯誤。
程式方面的知識，可以成為解決問題的工具，像是用Python來自動化重複性的工作。
所以我覺得不管是哪方面都是很重要的。

```
